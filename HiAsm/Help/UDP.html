UDP

 UDP - компонент для работы с сокетами по протоколу UDP.

LocalPort - определяет порт, к которому будет привязан сокет. Иными словами любое другое приложение в сети сможет после создания сокета слать данные на этот порт, а компонент соответственно получить их одним из методов - doRecived или doRecivedFrom.

LocalHost - указывает то, к какому IP адресу будет привязан сокет. Это св-во задает фактически фильтр на принимаемые пакеты, который может быть настроен на прием данных только с локального компьютера, прием из сети и наконец прием всех пакетов, посланных на данный порт. Обычно это св-во остается пустым для приема всех пакетов.

RemotePort - номер порта удаленного компьютера. Именно в этот порт компонент будет отсылать данные при использование соответствующих методов.

RemoteHost - IP адрес удаленного компьютера для отправки данных.

AutoConnect - при установки этого св-ва в True компонент автоматически после создания совета привяжет IP адрес из св-ва RemoteHost к методу отправки данных doSend, в противном случае этот метод будет работать только для широковещательных пакетов. 

ReceiveMode - способ приема данных из сокета. При задание Thread - компонент автоматически создаст новый поток для прослушивания порта и будет вызывать событие onReceive всякие раз после прихода данных. При использование режима Manual вам необходимо вручную вызывать методы doReceive или doReceiveFrom для приема данных.
